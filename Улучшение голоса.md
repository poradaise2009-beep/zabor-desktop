# Рекомендации по улучшению звука и шумоподавления в Zabor

Для повышения качества передачи голоса и эффективности подавления фоновых шумов предлагается рассмотреть следующие технические решения.

## 1. Оптимизация кодека Opus (SDP Mangling)
Текущие настройки используют фиксированный битрейт и CBR. Оптимизация на уровне протокола позволит улучшить звучание без существенного роста нагрузки:
*   **Переход на VBR (Variable Bitrate)**: Opus гораздо эффективнее работает в режиме переменного битрейта. Это позволяет динамически выделять больше данных на сложные участки (взрывные согласные) и экономить на тишине.
*   **Увеличение максимального битрейта**: Для "студийного" качества голоса рекомендуется поднять `maxaveragebitrate` до **96-128 kbps**.
*   **Отключение DTX (Discontinuous Transmission)**: Установка `usedtx=0` в SDP предотвратит "проглатывание" первых слогов после пауз в речи.

## 2. Продвинутая цепочка обработки (Web Audio DSP)
Расширение использования `AudioContext` для создания полноценного тракта обработки:
*   **Noise Gate / Expander**: Вместо попыток «вырезать» шум из самого голоса (что часто портит тембр и создает эффект «подводного звука»), используется полное или частичное подавление звука только в те моменты, когда пользователь молчит. Это гарантирует 100% сохранение естественности речи.
*   **SpeexDSP (WASM)**: Более мягкий и классический алгоритм шумоподавления. Он создает меньше цифровых артефактов по сравнению с нейросетевыми решениями типа RNNoise, сохраняя "живой" тембр.
*   **High-pass Filter (80-120Hz)**: Включение фильтра высоких частот в AudioContext позволяет эффективно убрать низкочастотный гул (вентиляторы, вибрации), не искажая основной диапазон голоса.
*   **Dynamics Compressor**: Обязательный элемент для "выравнивания" звука. Он делает тихую речь разборчивой, а громкие возгласы — не бьющими по ушам собеседников.

## 3. Модернизация VAD (Voice Activity Detection)
Улучшение логики определения активности голоса для более корректной работы индикаторов:
*   **Расширение частотного диапазона**: Увеличение верхней границы фильтра (сейчас 4000Гц) до 8000Гц позволит лучше захватывать высокочастотные гармоники речи.
*   **Внедрение Гистерезиса (Hysteresis)**: Добавление времени удержания (Hold time ~200-300мс) после прекращения речи, чтобы индикатор "говорения" не мерцал во время коротких естественных пауз между словами.

## 4. Настройки захвата (MediaConstraints)
*   **Тонкая настройка AGC**: В случае нестабильной работы нативного Auto Gain Control (когда звук становится слишком тихим или громким произвольно), рекомендуется переход на софтовую реализацию AGC в составе `AudioContext`.
*   **Обновленные методы Echo Cancellation**: Использование дополнительных флагов Chromium (например, `googEchoCancellation2`) для более глубокого подавления эха в сложных акустических условиях.
