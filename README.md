# ZABOR

Десктопное приложение для голосового общения, построенное на гибридной P2P-архитектуре.

## 🏗 Архитектура системы

Проект базируется на паттерне **Monolithic Server + P2P Media**. 

* **Signaling Server & State Manager:** Единый процесс на ASP.NET Core управляет состоянием комнат, списками друзей и сигналингом (обменом SDP/ICE) через WebSocket (SignalR). 
* **Server-as-Truth / Optimistic UI:** Клиент использует `session.json` и оптимистичный рендеринг для мгновенного отклика (например, при входе в канал). Сервер проверяет эти действия и рассылает финальный подтвержденный стейт.
* **P2P Audio (Mesh WebRTC):** Голосовой трафик **никогда не проходит через сервер**. Клиенты устанавливают зашифрованное SRTP-соединение друг с другом, что обеспечивает масштабируемость и минимальную задержку.

## 🛠 Стек технологий

### Клиент (Desktop)
- **Оболочка:** Electron 33 (IPC-взаимодействие, работа с файловой системой)
- **UI:** React 18, Tailwind CSS, Lucide React
- **Стейт-менеджмент:** Zustand (слайсы для каналов, юзеров, модалок)
- **Голос:** WebRTC, Web Audio API, `@jitsi/rnnoise-wasm`

### Сервер (Backend)
- **Фреймворк:** C# ASP.NET Core 8
- **Реалтайм:** SignalR (с кастомными задержками реконнекта и Grace Period)
- **Данные:** SQLite, Entity Framework Core (персистентность) + In-memory ConcurrentDictionary (лайв-стейт: онлайн, каналы)

## 📡 Логика сервера

1. **Коммуникация:** Отсутствует классический REST API. Все взаимодействие клиент-сервер (разлогин, изменение профиля, ping/latency, WebRTC-обмен) идет через методы SignalR Hub'а.
2. **Точечные рассылки:** Для распределения стейта используются `Groups` в SignalR. Пользователи получают обновления статуса в зависимости от того, просматривают они список каналов или сидят внутри конкретного (через группы `View_{channelId}`).
3. **Достижения и Статистика:** Сервер собирает метрики (время в эфире, количество уникальных комнат, ночные звонки) полностью на своей стороне, избегая читерства с клиента.

## 🎧 Аудио Пайплайн

Сырой звук с микрофона проходит через кастомный DSP-пайплайн перед отправкой:
1. **ML-Шумоподавление:** RNNoise AudioWorklet отсекает клавиатуру и фоновый шум.
2. **Эквализация:** `BiquadFilter`'ы обрезают инфранизкие частоты (убирают вибрации) и корректируют сибилянты.
3. **VAD (Voice Activity Detection):** Собственный алгоритм в time-domain отслеживает двойные пороги уровня громкости для определения речи и включения индикаторов.
4. **Кодек Opus:** Сильно оптимизирован через SDP Munging (48kHz, 96 kbps, FEC, DTX).

## 📥 Установка

Скачайте инсталлятор последней версии со страницы [Releases](https://github.com/poradaise2009-beep/zabor-desktop/releases).

> [!WARNING]
> **Windows SmartScreen** может заблокировать запуск, так как программа пока не имеет платной цифровой подписи.
> 
> **Для установки:**
> 1. Нажмите **«Подробнее»** (More info) в окне предупреждениящика.
> 2. Выберите **«Выполнить в любом случае»** (Run anyway).

## 🚀 Разработка клиента

```bash
# Установка зависимостей
npm install

# Dev-режим с hot-reload
npm run dev

# Сборка NSIS инсталлятора для Windows (x64)
npm run dist:win
```

*Бэкенд-часть компилируется отдельно через `dotnet publish` и деплоится за Nginx на Ubuntu.*
